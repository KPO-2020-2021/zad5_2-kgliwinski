Building

Build by making a build directory (i.e. build/), run cmake in that dir, and then use make to build the desired target.

Example:

> mkdir build && cd build
> cmake .. -DCMAKE_BUILD_TYPE=[Debug | Coverage | Release]
> make
> ./main
> make test # Makes and runs the tests.
> make coverage # Generate a coverage report.
> make doc # Generate html documentation.

Tests

Tests are made using doctests. There are two different ways to initiate tests:

> cmake .. -DCMAKE_BUILD_TYPE=[Debug | Coverage | Release]
> make
> ./unit_tests

OR:

> cmake .. -DCMAKE_BUILD_TYPE=[Debug | Coverage | Release]
> make fulltest

Important note: there are problems while using make fulltest command. Tests that write data to *.dat* files will be shown as failed while using this command. In case *./unit_tests* is used, all tests have success status. The problematic tests have also been done manually and everything seems to work good.
